## Suggested Actions (prioritized)

### Priority 1 — Risk reduction (target: next sprint)

1. **Add a Coverlet/`coverage.runsettings` file** at the solution root
   and wire it into the `Doc/publish.ps1` flow so every CI run produces
   an HTML report (`reportgenerator`). This replaces the qualitative
   bands above with real numbers and makes regressions visible.
2. **DiffusionNexus.Civitai**: add unit tests around `CivitaiClient`
   using `HttpMessageHandler` mocks (System.Net.Http.HttpClientFactory
   testing pattern) and golden-JSON fixtures for each `Civitai*` DTO.
   Cover at minimum: success, 401, 404, paging, and rate-limit responses.
3. **DiffusionNexus.Inference**: add tests for
   `ComfyUiModelCatalog` and `StableDiffusionCppLoader` resolution
   logic (these are pure C# and easy to unit test). Wrap native
   `DiffusionContextHost` in an interface so the backend can be tested
   without loading the native DLL.
4. **DiffusionNexus.Captioning**: cover `ImagePreprocessor`
   (deterministic) and `CaptioningModelManager` (file-system layout,
   download path resolution). Use `IFileSystem` (System.IO.Abstractions)
   abstractions where missing.

### Priority 2 — Stability of core services

5. **DiffusionNexus.Service / ComfyUI**: extract `HttpClient` and
   process launching behind interfaces (already exist for some), then
   add tests for `ComfyUIReadinessService` (state machine),
   `ComfyUIFeatureRegistry` (mapping table) and `ComfyUIWrapperService`
   process lifecycle.
6. **DiffusionNexus.Service / Metadata**: tests for
   `ModelMetadataService`, `ModelDiscoveryService` (filesystem walk +
   pairing logic) and `ModelFileSyncService` — these are pure logic
   given a fake filesystem.
7. **DiffusionNexus.Infrastructure**: add tests for `TaskTracker`
   (concurrency / handle disposal), `UnifiedLogger` (level filtering &
   sink fan-out), `ActivityLogService` (subscriber notification order).

### Priority 3 — Persistence safety

8. **DiffusionNexus.DataAccess**: introduce an integration test fixture
   that spins up a SQLite (in-memory or temp file) `DiffusionNexusCoreDbContext`,
   applies all migrations, and verifies each repository
   (`AppSettings`, `DisclaimerAcceptance`, `InstallerPackage`,
   `ModelFile`) round-trips. This guards against migration breakage
   (per the repo guideline that `publish.ps1` must run before entity
   changes).
9. Add a smoke test that opens the SDK database
   (`diffusion_nexus.db`) read-only to confirm it remains separate from
   `Diffusion_Nexus-core.db`, preventing the documented mix-up.

### Priority 4 — UI logic safety net

10. **MainWindow & navigation**: write a `DiffusionNexusMainWindowViewModelTests`
    fixture covering: module navigation selection state, disclaimer
    gating (`IsDisclaimerAccepted`), hidden Diffusion Canvas toggle,
    and `IsMenuOpen` persistence.
11. **Settings & About**: add `SettingsViewModelTests` (validation,
    persistence calls) and `AboutViewModelTests` (version string).
12. **Tab VMs** in `ViewModels/Tabs/**`: target the ones that contain
    branching logic (Dataset Management, Bucket Analysis, Batch
    Crop/Scale, Batch Upscale, Captioning, Duplicate Detection).
13. **DiffusionCanvas**: even though hidden, `DiffusionCanvasViewModel`
    + `GenerationFrameViewModel` orchestrate background work and should
    have at least lifecycle tests before being made visible.

### Priority 5 — Tooling & process

14. Adopt a **coverage gate** in CI (e.g. fail if line coverage of
    `DiffusionNexus.Service`, `DiffusionNexus.Domain`,
    `DiffusionNexus.DataAccess`, `DiffusionNexus.Civitai`,
    `DiffusionNexus.Captioning`, `DiffusionNexus.Inference` drops below
    an agreed baseline — start at the current measurement, then ratchet).
15. Per the repo `copilot-instructions.md`: every new bug fix should be
    preceded by a regression test. Enforce via PR template / checklist.
16. Grow `DiffusionNexus.IntegrationTests`: at minimum add one
    end-to-end test per "module" reachable from the main window
    navigation (smoke level — open module, verify ViewModel resolves,
    no exceptions).
17. Consider splitting `DiffusionNexus.Tests` into per-area test
    projects (e.g. `*.Service.Tests`, `*.UI.Tests`) so that
    targeted runs and per-project coverage gates become easy.
